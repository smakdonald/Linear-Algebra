It is natural to turn the proof of [cross-reference to target(s) "thm:form_diag" missing or not unique] into an algorithm that starts with a basis for \(V\) and converts it into a basis such that \(B\) has a diagonal matrix. One step of the algorithm is the following
\begin{equation*} \set{\vs} \ \longrightarrow\ \set{\v_1,\v_2-\frac{B(\v_1,\v_2)}{B(\v_1,\v_1)}\v_1, \v_3-\frac{B(\v_1,\v_3)}{B(\v_1,\v_1)}\v_1, \dots, \v_n-\frac{B(\v_1,\v_n)}{B(\v_1,\v_1)}\v_1}. \end{equation*}
Unfortunately this process only works if you can guarantee that at least one of the \(\v_i\) has \(Q(\v_i)\not=0\text{.}\) Sadly there are examples where \(Q(\v_1)=Q(\v_2)=\dots=Q(\v_n)=0\) but \(B\not\equiv0\text{.}\) For instance if \(Q((x,y)) = 2xy\) then \(Q((1,0))=Q((0,1))=0\text{,}\) but \(Q\) is not identically \(0\) on \(\R^2\text{.}\) One situation where this algorithm does work is when \(Q\) (or equivalently B) is positive definite, meaning that for all \(\v\not=0\) we have \(Q(\v)>0\text{.}\) In this context this algorithm is known as the Gram-Schmidt algorithm.
