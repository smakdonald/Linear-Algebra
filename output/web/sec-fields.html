    Section 2.1 Fields

    We start by discussing where our scalars can come from. Scalars are the numbers that we use to scale vectors. So far we have used only real numbers for our scalars. But there are other possibilities.

    Definition 2.1.

    A field is a collection of objects (that we think of as numbers) that satisfy the same algebraic properties as \(\R\) with respect to multiplication and addition.
    Here then is an explicit list of what we require in order to say that a collection of numbers, equipped with definitions of \(+\) and \(\times\text{,}\) is a field:
    A field is a set \(\F\) (that we think of as numbers) together with definitions of \(+\) and \(\times\) (in other words, together with functions \(+,\times:\F\times \F \to \F\)) such all the following properties hold:
    1. Addition is commutative: for all \(x, y\in \F\) we have \(x+y=y+x\)
    2. Addition is associative: for all \(x, y, z\in \F\) we have \(x+(y+z) = (x+y)+z\)
    3. There is an additive identity: there is a special number \(0\in \F\) such that for all \(x\in \F\) we have \(0+x = x\)
    4. There are additive inverses: for all \(x\in \F\) there is an element that we denote \(-x\) such that \(x+(-x)=0\text{.}\)
    1. Multiplication is commutative: for all \(x, y\in \F\) we have \(x y=y x\)
    2. Multiplication is associative: for all \(x, y\in \F\) we have \(x(y z)=(x y)z\)
    3. There is a multiplicative identity: there is a special number \(1\in \F\) such that for all \(x\in \F\) we have \(1x = x\)
    4. There are multiplicative inverse for non-zero numbers: for all \(x\in \F\wo\set{0}\) there is an element that we denote \(x\inv\) such that \(xx\inv=1\)
    If you have seen such things before (and do not worry if you haven't) we're just stating that \((\F,+)\) and \((\F\wo\set{0},\times)\) are Abelian groups.
    1. Multiplication distributes over addition: for all \(x, y, z\in \F\) we have \(x(y+z) = x y+x z\text{.}\)
    2. \(\displaystyle 1\not = 0\)

    Remark 2.2.

    The fact that the operations of addition and multiplication are functions \(+,\times:\F\times \F \to \F\) implicity gives the following additional properties:
    1. Closure under addition: for all \(x, y\in \F\) we have \(x+y\in \F\text{.}\)
    2. Closure under addition: for all \(x, y\in \F\) we have \(x\times y\in \F\text{.}\)
    The most important examples of fields are \(\R\) and \(\C\) but there are others that we will seriously consider.

    Example 2.3.

    The complex numbers are the numbers of the form
    \begin{equation*} \C=\{a+bi \mid a,b\in \R\} \end{equation*}
    with addition and multiplication defined by
    \begin{equation*} \begin{aligned} (a+bi)+(c+di) &= (a+b)+(c+d)i\\ (a+bi)\times(c+di) &= (ac-bd)+(ad+bc)i \end{aligned} \end{equation*}
    Note that by our definition of multiplication
    \begin{equation*} i^2=i\times i=(0+1i)\times(0+1i)=-1+0i=-1. \end{equation*}
    We can think of \(\R\) as a subset of \(\C\) by identifying \(r\in \R\) with \(r=r+0i\in\C\text{.}\) This makes \(\R\) as subfield of \(\C\text{.}\) A subfield is a subset of a field which is itself a field with respect to the operations on the larger set.

    Example 2.4.

    The rational numbers \(\Q\) are a field, indeed they are a subfield of \(\R\) (simply meaning that \(\Q\of \R\) and the field operations for \(\Q\) are the same whether you think of elements of \(\Q\) as rational numbers or real numbers). This fact means that conditions A1-A2, M1-M2, and C1 are trivially satisfied, since they're true for real numbers, not just rationals. Since the special real numbers \(0,1\) are in fact in \(\Q\) we also get A3, M3, and C2 for free. This only leaves A4 and M4; given \(x\in \Q\) we know that there are real numbers \(-x\) and \(x\inv\) such that \(x+(-x)=0\) and \(xx\inv=1\text{,}\) but what we need to know is that there are rational numbers with these properties ‚Äâ1‚Äâ. Fortunately it is that case that if \(x\) is rational then so are \(-x\) and \(x\inv\text{.}\) Thus \(\Q\) is a field.

    Example 2.5.

    The field with two elements, \(\F_2\text{,}\) has as its only elements the two residue classes modulo \(2\text{,}\) with addition and multiplication defined modulo \(2\text{.}\) To be specific, we have the following addition and multiplication tables.
    \begin{equation*} \begin{array}{c|cc} + & 0 & 1 \\ \hline 0 & 0 & 1 \\ 1 & 1 & 0 \\ \end{array} \qquad \begin{array}{c|cc} \times & 0 & 1 \\ \hline 0 & 0 & 0 \\ 1 & 0 & 1 \\ \end{array} \end{equation*}
    There are in fact many interesting examples of vector spaces where we use \(\F_2\) for our numbers. For instance many schemes of data transmission use the ideas of linear algebra to build error correction into digital communication.

    Definition 2.6.

    We make some standard abbreviations when computing in fields. We write
    \begin{equation*} \begin{aligned} x-y &\defeq x+(-y) \\ x/y &\defeq x y\inv \\ x^0 &\defeq 1 \\ x^n &\defeq \underbrace{x \times x \times \dots \times x}_{\text{$n$ factors}}\quad \text{for integers $n\ge 1$} \\ x^{-m} &\defeq \underbrace{x\inv \times x\inv \times \dots \times x\inv}_{\text{$m$ factors}} \quad \text{for integers $m\ge 1$} \end{aligned} \end{equation*}
    We also define, for \(n\in \N\text{,}\)
    \begin{equation*} n \defeq \underbrace{1+1+\dots+1}_{\text{$n$ summands}} \in \F. \end{equation*}

    Remark 2.7.

    Above we have defined elements \(1,2,3,\dots\) in any field \(\F\text{.}\) In \(\Z_7\) we have \(10=3=-4\text{,}\) and in \(\Z_{29}\) we have \(59=30=1\text{.}\)
    You might think it would be confusing that \(287\) is simultaneously the name of an integer and a field element, particularly in fields \(\F_{7}\) or \(\F_{41}\) where \(287=0\text{,}\) but context resolves all such confusions, usually without anyone noticing.
    The following rules of arithmetic hold in any field.

    Lemma 2.8.

    If \(\F\) is a field an \(x, y\in \F\wo\set{0}\) then for For all \(n, m\in \Z\) we have
    \begin{equation*} \begin{aligned} (x y)^n &= x^n y^n & (x/y)^n &= x^n y^{-n} \\ x^n x^m &= x^{n+m} & \left(x^n\right)^m &= x^{nm}. \end{aligned} \end{equation*}
    Of course if \(x=0\) we don't need to find a multiplicative inverse for \(x\text{.}\)
